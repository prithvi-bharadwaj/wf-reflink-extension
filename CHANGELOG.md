## How to maintain this file

When you ship something:

1. Add a bullet under `## [Unreleased]`, picking the right section header:
   - **Added** — new features
   - **Changed** — behavior of existing features
   - **Deprecated** — soon-to-be-removed
   - **Removed** — gone
   - **Fixed** — bug fixes
   - **Security** — vulnerability patches
2. When you cut a release:
   - Rename `## [Unreleased]` to `## [X.Y.Z] - YYYY-MM-DD`.
   - Add a fresh empty `## [Unreleased]` section above it.
   - Bump `version` in `package.json` to match.
   - `git tag vX.Y.Z && git push --tags`.
   - `npm run release` and upload `dist/RefLink-arm64.zip` to the GitHub release.

One bullet per shipped behavior. Lowercase. Same voice as commit messages.
Skip internal refactors and test-only commits unless they change behavior.

## [0.1.0] - 2026-04-25

First working version. Menu-bar tray app for macOS that injects clipboard
references into Wispr Flow dictation.

### Added

- electron tray app (no Dock icon, login-item auto-start)
- auto-capture on Wispr Flow hotkey (hold or toggle mode, configurable)
- reference queue: every `Cmd+C` during a session becomes a numbered ref
- trigger phrases (`inserted/embedded/insert/embed` + `link/image`)
- custom trigger-phrase editor window
- two paste modes: segmented (inline text + images) and batch (text first, images appended)
- tunable paste delay for slow apps (Notion, etc.)
- pre-paste `Cmd+Z` to undo Wispr's original insert before segmented re-paste
- first-run welcome tour (onboarding overlay)
- floating queue counter pill anchored near Wispr Flow
- bracket-dot template tray icon `[ • ]` with three states: idle (no dot), waiting (pulsing dot), active (solid dot); auto-tints for light/dark menu bars
- app icon (.icns) generated by `scripts/make-icon.js`, baked in by `npm run build`
- `npm run release` produces `dist/RefLink-arm64.zip` for distribution
- 32 regression tests covering reference queue, replacer, transcription detector, trigger regex
